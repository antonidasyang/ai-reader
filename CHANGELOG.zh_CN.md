# AI Reader 更新日志

## v1.0.0 — 2026-06-29

云端版。v0.2.0 让 AI Reader 能在单台机器上服务真实用户;v1.0.0 让
一个团队跨机器共享同一个文献库 —— AI Reader 从本地阅读器变成了
协作式文献工作台。

### 云端文献库
- **以课题组织共享库。** 按研究课题归类文献,每个课题是一个云端
  同步的文献库,拥有各自的成员。
- **一键加文献并自动补全题录。** 打开 PDF 点 **+ 添加**:题名 /
  作者 / 年份 / 出处 会按 DOI 或 arXiv 号自动补全,PDF 同时上传
  (按内容寻址,同一文件全课题只存一份)。
- **整库全文检索** —— 题名、摘要、作者、标签。
- **离线优先同步。** 断网也能浏览、编辑;联网后自动对账(WebSocket
  实时,轮询兜底),按字段后写胜。

### 协作
- **多人课题**,owner / editor / viewer 三种角色,按邮箱邀请成员。
- **共享 AI 解读。** 把 AI 摘要 / 解读共享到课题,同伴在 **共享**
  里即可看到。

### 账号与更新
- **CAS 单点登录。** 浏览器一键登录,应用内不保存密码。
- **自建自动更新。** 启动时检查服务端版本清单,有新版一键下载。

### 界面
- **文献栏并入主工具栏** —— 课题选择、成员、账号不再独占一行。
- 以上功能**全部中文化**。

## v0.2.0 — 2026-05-02

打磨与发布版。v0.1.0 是第一个能跑起来的版本;v0.2.0 是第一个能在
开发者机器之外的真实环境中正常运行的版本。

### 对话
- **每篇论文支持多会话对话。** 每篇论文在对话面板顶部维护自己的
  会话标签栏:**+** 新建,**×** 关闭,双击重命名。
  会话标题在累计三轮对话后从首条用户消息中自动生成。会话按论文
  保存到 `<AppData>/cache/chat/<paperId>.json`。
- **类型化消息渲染。** 已完成的回复以一组类型化 QML 项渲染
  (文本、代码、数学公式),不再统一塞在一个 `TextEdit` 中。
  代码块显示语言标签和复制按钮;数学公式(`$$ … $$`)通过
  MicroTeX 渲染,失败时回退为黄底原始 LaTeX。
- **段落级翻译** 加入段落右键菜单 —— 不必重跑全文翻译就能单独
  翻译一段。

### 段落
- **段落级原文/译文显示开关。** 每段单独切换,持久化到段落缓存。
- **设置 → 字号(像素)** 可分别设置「章节目录 / 论文解读 /
  段落 / 对话」面板的正文字号。各面板内的标题与标签会按比例
  放大,保持视觉层级。
- **目录与解读不再被段落编辑清空** —— 当你拆分、合并、删除单段
  时,清理与重载的逻辑只在真正切换论文时触发。

### 界面与新手引导
- **首次启动新手向导** —— 半透明遮罩配合工具栏按钮的高亮
  spotlight。任何时候都可以从工具栏的 **?** 重新打开。
- **面板布局与尺寸会被记住。** 拖动任意面板左上角的 **⋮⋮** 拖柄
  来重新排列;拆分条的位置和顺序都会持久化。
- **流式输出时自动滚动到底部。** 对话与解读面板都支持。向上滚
  动会暂停自动跟随,这样阅读旧内容时不会被强制下拉。

### 更新与隐私
- **应用内更新检查。** 启动时拉取 GitHub Releases 上的小型
  `manifest.json`,有新版会在窗口底部显示蓝色横幅,带「下载」
  按钮。可在「设置 → 更新与隐私」中关闭。
- **崩溃报告(可选加入)** 通过 Sentry-Native(构建时需配置
  `-DAIREADER_ENABLE_SENTRY=ON`)。默认关闭;不收集任何 PII。
- **应用版本号显示** 在设置对话框页脚和安装包的 PE 资源元数据
  中(右键 → 属性 → 详细信息)。

### 打包
- **Windows 安装包流水线。** Build → `windeploy.bat` → Inno
  Setup `AiReader.iss` → 可选 `sign-windows.ps1` →
  `publish-release.ps1`。bat 会自动通过 `build\CMakeCache.txt`
  定位 `windeployqt`;部署会附带 MSVC 运行时 DLL
  (vcruntime140.dll 等),没装 Visual C++ 可再发行包的用户也能
  正常启动。
- **持久化启动日志** 位于 `<AppData>\AI Reader\launch.log`,
  方便诊断 GUI 子系统下吞掉 stderr 的启动失败。
- **MicroTeX 字体路径可重定位** —— 数学公式现在能在已打包的安装
  中正常渲染,不只在 `build/` 下的开发版可用。
